A brief look at the "map" method (and "each" too!)

Monday, February 22, 2016

When delving a little further into Ruby, you will come across a couple of very useful data structures and the understanding of which will be immensely useful to you in your future as a Ruby (or any other language, really) programmer. I'm talking about arrays and hashes. I discussed the basics of these two in my last post but today I want to get a little more into how to manipulate the data that these structures contain.

So, you know how to create arrays and hashes now and you know how to put some data into them. But how do you use the data that you've stored? Well, all objects in Ruby (and EVERYTHING in Ruby is an object. Really. Everything.) have what are called built-in methods. Methods are basically little routines that you can call upon your object to perform some pre-specified task on said object. There are many that are common to all objects and others that are common to only certain types of objects. Today I'll cover a couple, but one in particular: map.

However, before we jump to the discussion of the map method, I would like to build up to that by touching on the each method. each allows you to iterate (which is another way of saying of doing something repeatedly) over an array or hash, going over each element that you have stored in it. A method call for each would look something like this: 

        
          new_array = Array.new
          new_array.each do |x|
            # your code goes here
          end

What our example shows is the creation of a new array called (you guessed it) new_array. We are then calling the method each on new_array. As you can see, you do this by using a "." to let new_array (or another object) know that it is being told to perform the method on the right, each. Now, along with each is a little bit of setup: you have do |x| next to it, which tells each to "do (x) action" to whatever is contained in new_array. That action is determined by the code block of your design on the following line(s). As with a lot of things in Ruby, don't forget to finish your code block with the word end. This method works the same for hashes as well, but instead of working with one variable, you are working with two: one for your keys and one for your variables.

So, let's talk about map. map is very similar to each in that it also iterates over the array/hash but where it differs from each is that after iterating through the array/hash and performing it's code on each element, it then outputs the result into a new array. Pretty nifty, huh? The setup looks very similar, except we're going to give map it's code block a little differently: 

        
          new_array = [1, 2, 3, 4]
          new_array.map {|x| x + 1}

In this example, I've already put four elements into our new_array and then in the next line, I'm telling it use the map method to perform the code block {|x| x + 1|}. This means that for each element, map will add 1 to it, then output the result into a new array which would look like this: 

        
          [2, 3, 4, 5]

Simple enough. Now, as a bit of a warning (and also if you ever need to use it) map has a destructive sibling, map!. "Destructive" is a bit of a tricky term in Ruby as it can mean slightly different things in different places. This usage here is of the more drastic variety. If you were to call map! in the above example as opposed to map, it would permanently change the contents of new_array and not output the result of the code block into a separate array. This can be useful at times, but also a little confusing and potentially catastrophic when you're just starting out.

Again, I hope that this was a helpful introduction to the each and map methods. I highly recommend toying around with these (and other methods) in irb to get a better feel of how these work. Then you can try them out in your own code!
