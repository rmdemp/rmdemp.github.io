JavaScript vs. Ruby, Round 1

A brief comparison of Ruby Classes and JavaScript Constructor Functions

Sunday, March 6, 2016

Welcome back. Today we'll be shifting gears yet again and discussing some fundamental differences between Ruby Classes (which I introduced in the post before this) and a very new concept to me: JavaScript Constructor functions. This will be just as much a learning opportunity for you as it will be for me.

So we already know that Ruby classes basically act as blueprints for creating new objects of the given class. JavaScript constructor functions work roughly the same way. Functions in JavaScript are blocks of code that perform specific tasks much akin to methods in Ruby. To begin, let's look at how you create a constructor function: 

        
          function newObject() {
            // code here
          };

Pretty easy so far. Now when we start adding properties to this new function we will begin to see where constructors differ from regular JavaScript functions.

        
          function newObject(argument) {
            this.property1 = argument
            this.property2 = "property 2"
            this.property3 = function() {
              // code block here
            };
          };

As you can see, we now have three properties inside our function: a variable defined with whatever argument you input, a string, and yes, even a function within our function! Now, you're probably wondering what is up with "this". Here is where we will see similarities between (and differences from) Ruby classes.

"This" is self-referential. It refers to the properties inside this specific instance of this function. Very similar to instance variables and methods within Ruby classes. Here is another similarity for you: when you want to create a new instance of a function, you must instantiate it thusly:

        
          var newNewObject = new newObject();

Look familiar?

All of this is somewhat relatable territory between the two languages. However, this is where things also start to differ. In Ruby, when you create a new instance of a class, all of it's methods and variables are the same for each other instance you create for the same class (unless you alter the class itself.) Altering the class would cause all instances to change as well. This is not the case in JavaScript and is one of the reasons why constructor functions are so useful. You can use constructor functions when you need multiple instance of the same object. If you instantiate a new version of the function, you can change that individual instance's properties without affecting the properties of any instances of the same function. Pretty darn cool, eh?
