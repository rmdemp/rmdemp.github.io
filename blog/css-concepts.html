Positioning in CSS: A Very Brief Overview

What are the differences between static, relative, absolute, and fixed positioning?

Sunday, February 7, 2016

Ah, yes. Positioning in CSS. You can add borders, change colors and fonts, and even make your links do cool stuff with ease. Positioning shouldn't be any different, right? It should only take a little while. Let's get to it! (Fast forward four hours later...) After much trial and error (and more than a little frustration), you finally got your navigation bar setup like you want it! Now just got place the rest of the elements on the page...

The above paragraph isn't too far from reality for most of us (present company included.) What makes positioning in CSS often such a frustrating task? I have this element in place now why are all the rest of them wonky?! In this entry, I want to try to help ease this pain a little bit by delving briefly into the differences between the various forms of positioning available in CSS.

Each HTML element can be moved around the screen by using the "position" attribute in CSS. This attribute has six different property values, the main four which we will cover here are: relative, absolute, fixed, and static. Each of these has their different uses and ablities and here they'll each be broken down a little for you.

The default value for an element is "static" which means just that: it appears on the page in normal flow, one after another down the page vertically. This could be used for headers, paragraphs, and other text that needs to remain fairly uniform on the screen.

Relative positioning is a little bit more dynamic. This means that an element is placed relative to it's regular position in normal HTML flow. You can move an element with a position value of relative by altering the margin and padding attributes. This will allow you to offset text, images, or other elements in regards to their surrounding elements, which can be used as sidebars or the like.

A little more tricky is absolute positioning. When an element is given this "position" value, it is removed from normal flow and is positioned in relation to the element that contains it. This will not affect the position of any of the surrounding elements. A good example of this is text within a div: the text can be moved about within the div without disturbing the position of the content of the element next to it.

And finally, there's fixed positioning. This is a little similar to absolute positioning but here the element is positioned relative to the browser window as opposed to its containing element. This is useful for a toolbar or navigation bar that you want to have visible as you scroll through the content of a page.

As I said, this was a very brief overview of the different values for the position attribute but I hope that it shed a little light on this tricky subject.
