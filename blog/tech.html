Regular Expressions

A very brief introduction to syntax

Saturday, March 13, 2016

Today's post is going to cover another topic that is rather new to me and one which I find very interesting in my budding career as a developer. I'll be briefly discussing regular expressions (if you couldn't tell by the title)and introducing some of their syntax.

According to the site Regular-expressions.info, "a regular expression (regex or regexp for short) is a special text string for describing a search pattern." These text strings can be as short as a single character or (to use another of their examples) as long or longer than this: 

        
          \b[A-Z0-9._%+-]+@[A-Z0-9.-]+\.[A-Z]{2,}\b

(That, by the way, is a regexp used to find any email address in a block of text.)

As you can see, there are a lot of other symbols mixed in here as well. They aren't all as straightforward as the seem. These are referred to as metacharacters and provide special functions within the syntax of the regexp. There are twelve total: 

        
          \ ^ $ . | ? * + ( ) [ {

As with Ruby, if you wanted to use any of these characters normally, you would have to "escape" that character using the "\" symbol. Many programming languages use regular expressions and have different methods for implementing them so please consult the appropriate documentation to see how to use these within a given language.

Regular expressions also use what are known as character classes which will search for specific characters. A common example would be gr[ae]y which would return matches for "gray" and "grey" and not "graey". Letter order is not specific here. Hyphens can be used within a character class (i.e. [0-9]) to search through a range and will only return one character within that range. Dots will match a single character (i.e. gr.y will match grey, gray, gr%y.) The caret (^) and $ symbols match at the beginning and end of strings respectively. The | symbol can be used in Boolean ("or" searches.) The ? symbol makes previous characters optional matches, the * symbol matches preceding characters zero or more times, and the + symbol matches preceding characters one or more times.

Today's entry is meant to be a very brief overview into the world of regular expressions. I do plan on revisiting this subject in future posts as I learn more in order to "spread the wealth." Regular expressions are vast and powerful tools with many, many uses and are more common than one might think. Until next time, I recommend trying out the tutorials on Regular-expressions.info in order to get better practical understanding of the subject and also visit the Wikipedia page for a more in-depth look at the history and structure of regular expressions.
